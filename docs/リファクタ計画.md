# main.js / renderer.js モジュール分割 計画

`main.js`(約1300行) と `renderer.js`(約1700行) が肥大化している。回帰リスクを抑えるため
**一度に全部やらず、純粋・自己完結な部分から少しずつ切り出して都度テストで裏取り** する方針。

## 済（この回で実施）
- `src/vault.js` — マスターパスワード暗号（新規＋`test-vault.js`/`test-vault-flow.js`）
- `src/sshconfig.js` — `~/.ssh/config` パーサ（`main.js` から抽出＋`test-sshconfig.js`）
- `src/ssh-algos.js` — レガシー暗号アルゴリズム一覧（純粋データを抽出）

E2E（`e2e/smoke.spec.js`）で「起動・主要UI・シェルタブ生成」を自動確認できるようになったので、
以降のレンダラ側の抽出も **E2E＋回帰テスト** で安全に進められる。

## 次の候補（安全度が高い順）
1. **main: IPC ルーティングの分割** — `fileserver:*` / `netscan:*` / `config:*` は既に別モジュールだが、
   `ipcMain.handle` の登録が main.js に散在。`src/ipc/*.js` に `register(ipcMain, deps)` 形式で束ねる。
2. **main: 接続系の分離** — `sshConnect`/`telnetConnect`/`serialConnect`/`shellConnect` を
   `src/connections/*.js` へ。`conns` Map と `handleIncomingData` を注入する形にする。
   ⚠️ `legacy-dh` を `ssh2` より先に require する **ロード順序** は厳守（壊すと旧暗号機器に繋がらない）。
3. **renderer: 機能単位の分割** — `renderer.js` から SFTP / 監視 / 転送UI / マクロ / 分割ペイン を
   `src/renderer/*.js` へ。`features3.js` と同様に index.html で読込順を管理。

## 進め方（各ステップ共通）
1. 純粋関数・データから切り出す（状態を持つものは依存を引数注入に）
2. `npm run check`（構文）→ `npm test`（回帰）→ `npm run e2e`（起動/UI）を必ず通す
3. 1コミット=1モジュールに保ち、問題時に戻しやすくする
